# QGC-маршрут для патрулювання бiля Лимана

Цей файл використовується у двох сценарiях:

- маршрут QGC у режимi **Auto** для перевiрки SITL та руху ровера на картi;
- файл маршруту для готового `auto_stub` у C2-сценарiї режиму **Guided**.

Маршрут можна змiнювати в QGC i зберiгати назад у цей `.plan` файл. Пiсля
змiни файлу маршруту перезапустити `auto_stub`, щоб вiн перечитав маршрут.

## Як запустити

1. Запустити SITL:
```bash
cd repository/homework_12
docker compose -f sim/compose.sitl.yml up -d
```

2. Вiдкрити QGroundControl i дочекатися пiдключення до ровера.
3. Перейти в **Plan** view.
4. **File** -> **Open...** -> вибрати `qgc/lyman-patrol.plan`.
5. Натиснути **Upload**.
6. Перейти в **Fly** view.
7. Для перевiрки прямого маршруту QGC зупинити `c2_service` та `auto_stub`, якщо вони запущенi.
8. Перевести ровер у **Auto**, натиснути **Arm**, потiм **Start Mission**, якщо QGC показує окрему кнопку старту.

## Як запустити через C2 / Guided

1. Запустити SITL.
2. Запустити бортовий стек.
3. У QGC перевести ровер у **Guided**.
4. Натиснути **Arm**.
5. Дивитися логи:
```bash
cd repository/homework_12/edge
docker compose up -d --build
docker compose logs -f auto_stub c2_service
```

У цьому сценарiї **Start Mission** не натискати: маршрут читає готовий `auto_stub`, а C2 передає точки маршруту в FC тiльки у станi `ARMED_GUIDED`.

`auto_stub` не читає телеметрiю з FC. Якщо перейти в **Manual**, поїхати джойстиком в iнше мiсце, а потiм повернутися в **Guided**, C2 має знову передавати поточну точку маршруту вiд `auto_stub`. Ровер може поїхати напряму до цiєї точки з поточної позицiї. Щоб почати маршрут з першої точки:

```bash
docker compose restart auto_stub
```

## Що всерединi

- 11 точок у районi Лимана.
- Тi самi точки у зворотному порядку.
- `DO_JUMP` на першу точку з repeat `-1`, щоб маршрут повторювався безкiнечно.

QGC **Auto** сценарiй обходить C2: QGC завантажує mission прямо в ArduPilot. C2-сценарiй використовує той самий `.plan` файл, але маршрут читає готовий `auto_stub`.
